Feedback Page Validation

The Feedback page was validated using the W3C Markup Validation Service. No errors were found, and only minor accessibility warnings were issued.

Reflection:

Ensuring all inputs had correct label associations and proper HTML5 syntax helped pass the validator easily. This reinforced the value of writing clean, semantic HTML from the start.

Team Page Validation

This page was validated successfully with all structural and accessibility standards met. ARIA roles were reviewed and adjusted to avoid deprecated values.

Reflection:

Initial validation failed due to incorrect use of role="listitem". After reviewing the standard, I replaced it with correct HTML semantics, which resolved the issue.

Content Page ST3 Validation

The SDG 6 content page passed all HTML5 validation checks. Structure and internal linking were verified.

Reflection:

One challenge was ensuring that internal anchor links were valid and uniquely identified by IDs. I learned the importance of matching href="#section-id" with a corresponding element in the document.
