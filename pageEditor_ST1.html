Splash Page

Technical Description

The Splash Page uses semantic HTML5 elements, including header, main, and section tags, to create a minimalistic and visually engaging introduction to the website. A loading spinner is implemented using a div tag with the class loader, styled in CSS with border properties to create a spinning effect via the keyframes spin animation, running infinitely . The background image (cleaninghands1.jpg) is applied with CSS background-size: cover and background-position: center to ensure full coverage and proper alignment. The h1 and h2 elements are styled with the Poppins font and a blue color for visual hierarchy, while the ul element lists team members without bullets (list-style-type: none). The layout is centered using text-align: center and margin: auto, ensuring a clean and welcoming design. The page is lightweight to ensure fast loading, aligning with its role as an entry point.

Accessibility

The Splash Page uses semantic HTML to provide a clear structure for screen readers. Descriptive text in the h1 and h2 elements aids navigation for assistive technologies. The viewport meta tag is implied (though not shown in the provided code) to support responsiveness, making the page accessible across devices. The loading spinner’s animation is visually distinct but does not rely on color alone, ensuring it is perceivable by users with visual impairments. Adding aria-label to the loader could further enhance accessibility for screen reader users.

Volunteer Page

Technical Description

The Volunteer Page employs semantic HTML5 elements like header, nav, main, and footer to structure content promoting volunteer opportunities. The nav element uses flexbox (display: flex) for responsive navigation link alignment, consistent with the site’s global style.css. Each card includes an img, h3, p, and button, styled with border-radius and box-shadow for a modern look. A feedback form uses a textarea with resize: vertical and star rating span elements for interactivity. The background image (volunteers.png) is overlaid with a semi transparent layer to improve text readability. CSS transitions (transition: 0.3s ease) on buttons and links enhance interactivity.

Accessibility

Semantic HTML ensures a clear structure for screen readers, with h1, h2, and h3 elements providing logical headings. The navigation links and form elements are keyboard accessible, with visible focus states via hover and focus styles. High-contrast text (e.g., white and blue on semi-transparent backgrounds) meets contrast standards. The viewport meta tag ensures responsiveness, and the card layout adapts to smaller screens via horizontal scrolling. The form’s textarea includes a placeholder for guidance, and the star rating system uses clickable span elements, though adding aria labels would improve screen reader support. Descriptive link text (e.g., “Join Now”) aids assistive technologies

Content Page

Technical Description

The Content Page (content_ST1.html) uses HTML5 section elements to organize content about SDG 6, with internal anchor links (a href="section-id") for smooth in-page navigation. The nav element for section links is fixed at the top with position- fixed and styled with background-color for visibility. Flexbox is used in the text-image-row and numbers-grid containers to create responsive layouts for text and statistics. The background image (cleaninghands1.jpg) is styled with background-size- cover and background attachment: fixed for visual appeal. CSS properties like border-radius and box-shadow enhance the modern aesthetic of sections and images. The stat-block elements use bold typography (font-size: 3rem) and color classes (e.g., .blue) for emphasis. Media queries adjust padding and navigation stacking for smaller screens, ensuring responsiveness.

Accessibility

Semantic section and nav elements provide a clear structure for screen readers, with h1, h2, and h3 headings defining content hierarchy. Anchor links are keyboard-accessible with clear focus states via hover styles. High contrast text (e.g., white and blue on dark or semi-transparent backgrounds) adheres to contrast guidelines. The responsive design, supported by the viewport meta tag, ensures usability across devices. Images in the.section-image container include alt text (assumed, though not shown in the code) for screen reader compatibility. The statistics section uses large, bold text for readability, but adding aria-describedby to stat blocks could further enhance accessibility for assistive technologies.

Challenges and Lessons Learned

A major challenge was aligning content responsively and keeping both navigation bars visible while scrolling. I solved this by using position: fixed and adjusting z-index. Another challenge was integrating responsive image layout, which was addressed with Flexbox. I also learned the importance of semantic HTML and testing for accessibility.

Compliance

My pages comply with Janet regulations by avoiding personal data collection, ensuring no copyrighted or uncredited content is used, and validating all HTML and CSS through W3C tools. I maintained secure linking and avoided any malicious scripts or popups.
