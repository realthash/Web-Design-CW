2. Technical Description

Feedback Page

This page was built using HTML forms for user input, including <input>, <textarea>, and <select> tags. CSS was used to validate fields and align the form responsively. Important features included required attributes for accessibility and JavaScript for submission validation. I styled buttons with transitions and shadows to enhance interaction.

Team Page

Built using Flexbox to create a responsive grid of team members. Each member profile includes a hover-reveal box using max-height transitions. Key HTML elements included <article>, <section>, and <ul>. CSS classes like .member and .details used border transitions, opacity effects, and shadowing to create interactive profiles.

Content Page (ST3)

Created an informative page titled “What You Can Do to Support SDG 6.” Key HTML structure includes semantic tags: <section>, <h2>, and <ul>. I used media queries to ensure the content displayed well on all devices. The design used a soft palette and emphasis classes like .highlight to separate impactful messages.

5. Challenges and Lessons Learned

One challenge was achieving responsive layout for the team profiles. I solved this by using Flexbox with flex-wrap and justify-content: center. Another issue was styling form elements consistently across browsers, which I fixed using reset styles and consistent font and padding rules. This project taught me the value of modular CSS and semantic HTML.

6. Accessibility

I ensured that all forms had labels connected to input fields using for and id attributes. Team profiles are navigable with keyboard using tabindex="0". I also used ARIA roles like role="listitem" and aria-labelledby for screen readers.

7. Compliance

All pages follow W3C validation standards and the UK’s Janet web publication regulations. Content is accessible, readable, and includes appropriate metadata. Images are optimized and hosted securely. The site does not include advertising, third-party cookies, or tracking scripts, ensuring user privacy and compliance.
